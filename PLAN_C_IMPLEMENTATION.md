---

## 🎯 目标

从当前的"定时扫描"模式升级为"实时触发"模式：

### 当前模式（方案 B）
```
钉钉发送视频 → 本地存储 → 等待扫描（60 秒） → 创建任务 → 分析 → 删除本地文件
```

### 目标模式（方案 C）
```
钉钉发送视频 → 本地临时存储 → 立即触发分析 → COS 永久存储 → 删除本地文件
```

---

## 📊 架构对比

| 特性 | 方案 B（当前） | 方案 C（目标） |
|------|--------------|--------------|
| **触发方式** | 定时扫描（60 秒） | 实时触发（立即） |
| **本地存储** | 临时（分析后删除） | 临时（分析后删除） |
| **COS 存储** | 永久 | 永久 |
| **延迟** | 60-120 秒 | <10 秒 |
| **改动范围** | 小（已实施） | 中（需修改渠道插件） |

#### 2.2 创建立即分析接口

**文件**: `ai-coach/immediate_analyze.py`（新建）

**功能**:
```python
async def analyze_video_async(task_id):
    """异步分析视频"""
    try:
        # 1. 上传 COS
        cos_url = await upload_to_cos(task_id)
        
        # 2. Qwen 分析
        result = await qwen_analyze(cos_url)
        
        # 3. 生成报告
        report = generate_report(result)
        
        # 4. 发送报告
        await send_report(report)
        
        # 5. 更新数据库
        update_task_status(task_id, 'completed')
        
        # 6. 删除本地临时文件
        cleanup_temp_file(task_id)
        
        logger.info(f"✅ 分析完成：{task_id}")
        
    except Exception as e:
        logger.error(f"❌ 分析失败：{task_id} - {e}")
        update_task_status(task_id, 'failed', last_error=str(e))
        # 保留本地文件，便于排查
```

**预计工作量**: 3-4 小时
